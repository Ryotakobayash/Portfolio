こんにちは。就活がひと段落しそうな修士1年ryota5884です。

今回は後輩への引き継ぎも兼ねて、このNUTMEGブログが生まれた背景や歴史、運営方法についてまとめてみようと思います。

## NUTMEGブログの目的

2021年の夏、NUTMEGでは団体の成長に伴い、以下のような考え・要望を持つようになりました。

- メンバーが就職活動をする際、どのような団体なのか簡単に紹介できるようにしたい
- 外部の人が団体に興味を持ったとき、活動内容・雰囲気を知ってもらえる場が欲しい
- メンバーがアウトプットの練習をする場を作りたい

2021年8月頃、当時のNUTMEG代表者から鶴の一声があり、上記のような状況を改善できるブログを作ろう！という運びになりました。

## 制作過程

学部1年の頃、修士2年の先輩から静的サイトジェネレーター「Hugo」の使い方を学んでおり、markdownで簡単に記事を作成できるサイトの作り方には心当たりがあったので、Hugoを使用して作ることに決めました。

> Hugoとは ──
> ウェブサイトを構築するための世界最速のフレームワーク
> Hugoは、最も人気のあるオープンソースの静的サイトジェネレーターの1つです。

## 技術構成

| 項目 | 使用したもの |
|------|-------------|
| 静的サイトジェネレーター | Hugo |
| Hugoテーマ | liva-hugo |
| ブログ記事の作成・管理 | HackMD |
| 文章の校閲 | VScodeプラグイン「テキスト校正くん」 |
| デプロイ | GitHub Pages |

## 運用方法

長く続けるためにあまり厳格にはせず、できる限りゆるくすることを意識していました。

| 項目 | 内容 |
|------|------|
| 記事の更新 | 定例MTでブログ発表後に投稿者が指名（テレフォンショッキング形式） |
| 著者名 | ペンネーム方式 |
| 記事のリジェクト | 基本なし |
| ネタがないとき | HackMDにネタ帳を作ってストック |

## 具体的な更新作業

1. `hugo new content post-yyyymmdd.md`でファイル作成
2. HackMDの記事内容をコピペ
3. 文書校正プラグインで誤字修正
4. `hugo`コマンドでHTML生成
5. GitHubにpushして自動更新

時間がかかるのは文章校正の部分です。ここに手間が掛からなければ5分程度で投稿は終了します。

## 今後の課題

- 現状では更新できるメンバーが少ない
- メンバーページの更新が少し手間
- コードを添付したときのレイアウトが醜い
- 著者名から過去記事を参照できるページが未実装

次の担当者が決まったら、恒常的な更新作業はお願いして改善してみたいなー。
