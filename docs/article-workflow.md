# 記事追加フロー・ガイドライン

このドキュメントでは、本ポートフォリオサイト（Astro + Content Collections）における新しい記事の追加方法、画像の管理方法、およびリモートリポジトリへ Push する前に行うべきことのチェックリストをまとめています。

---

## 1. 記事の追加方法

記事は Astro の **Content Collections** 機能を利用して管理されています。

### ファイルの作成
新しい記事を作成する場合は、以下のディレクトリに MDX (`.mdx`) ファイルを作成します。
画像等の自動最適化を活用するため、記事ごとのフォルダを作成し、その中に `index.mdx` を置く方法（または同名のフォルダに画像を置く方法）を推奨します。

**保存先:** `src/content/posts/`

**ファイル名の命名規則:**  
英数字とハイフンを用いたスラグ（URLの一部になります）を推奨します。  
例: `my-new-article.mdx`

### Frontmatter (メタデータ) の設定

ファイルの先頭に、以下の形式で Frontmatter を記述します。これは `src/content/config.ts` で定義されたスキーマに従う必要があります。

```yaml
---
title: "記事のタイトルをここに記述"
date: "2024-03-15"
excerpt: "（任意）記事の短い概要や抜粋を記述します"
tags: ["Astro", "React", "Tech"]
---

ここからマークダウン形式で記事の本文を記述します...
```

- **title** (必須): 記事のタイトル
- **date** (必須): `YYYY-MM-DD` 形式の公開日
- **excerpt** (任意): 記事一覧などで表示される抜粋文
- **tags** (任意): 関連記事の紐付け等に使われるタグの配列

## 2. 画像の管理方法

記事内で使用する画像は、自動的なリサイズ、軽量化（WebP変換等）、図表番号の自動付与の恩恵を受けるために、**MDX ファイルと同じ階層（相対パス）**で管理します。

### 執筆時の画像ペーストについて（推奨環境: VS Code / Cursor）
VS Code 等の IDE で執筆する場合、拡張機能 **Paste Image** (mushan.vscode-paste-image) の利用を推奨しています。（リポジトリの `.vscode/` に推奨設定が含まれています）

1. クリップボードに画像をコピーする
2. MDX ファイル上で `Cmd + Option + V` (Mac) または `Ctrl + Alt + V` (Windows) を押下する
3. 画像が自動的に `src/content/posts/記事名/` 配下（現在開いているファイルと同じ階層内）に保存され、同時に画像リンクが挿入されます。

### 手動で管理する場合の保存場所と参照方法
拡張機能を使わず手動で管理する場合は、記事MDXファイルと同じディレクトリ、またはその直下のフォルダに保存します。

**保存先例:** `src/content/posts/my-new-article/thumbnail.png`

記事内からは `./` 始まりの相対パスで参照します。

```mdx
<!-- MDXでの画像の埋め込み例 -->
![見えない代替テキスト(ALT)](./thumbnail.png "表示される図表タイトル(Caption)")
```

**新しい実装に基づく画像記法のルール:**
- `![ALT]` の部分: スクリーンリーダー等に読まれる**代替テキスト（見えない説明）**として保持されます。空にすることも可能ですが、アクセシビリティのため適切な説明を入れることを推奨します。
- `(URL "TITLE")` の `"TITLE"` 部分: 画面下部に表示される**図表タイトル**になります。

Astro と専用スクリプトにより、上記の画像リンクは以下のように出力されます。
- クライアントの画面幅に合わせたサイズの WebP 画像 (高速化)
- 縦長すぎる画像が画面を埋め尽くさないためのレスポンシブ CSS 適用 (`max-height: 60vh;`)
- JavaScriptにより自動的に `figure` タグで囲われ、`"図N: 表示される図表タイトル(Caption)"` のようなキャプション `<figcaption>` が付与されます。
- ※ `"TITLE"` を省略した場合、フォールバックとして `![ALT]` のテキストが図表タイトルとして表示されます。

## 3. Push する前にやること（チェックリスト）

変更をリモート（GitHub等）に Push し、Vercel でデプロイされる前に、ローカルで以下の確認を行ってください。

- [ ] **開発サーバーでの表示確認**
  ```bash
  npm run dev
  # または pnpm dev / yarn dev
  ```
  ローカルサーバー（通常 `http://localhost:4321`）を立ち上げ、以下を確認します。
  - トップページや記事一覧 (`/posts`) に正しく表示されているか
  - 記事詳細ページ (`/posts/[slug]`) でデザイン崩れがないか
  - 画像が正しく読み込まれ、表示されているか

- [ ] **Frontmatter の検証**
  - タイトルや日付のフォーマット、タグの指定が正しいか (`src/content/config.ts` の型制約に違反していないか)

- [ ] **プロダクションビルドのテスト**
  ```bash
  npm run build
  ```
  - デプロイ時のエラーを未然に防ぐため、ローカルでビルドが正常に完了することを確認します。

- [ ] **セルフレビュー**
  - 誤字脱字がないか
  - 個人情報や公開すべきでない機密情報（APIキーなど）が含まれていないか

すべて問題なければ、commit してリモートリポジトリへ Push します。Push 後は Vercel により自動でデプロイが走ります。
