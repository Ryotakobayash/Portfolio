## 現在の構成

<TwoColumn>
<div>

### Astro — 骨格と配信

- **Pages** — `src/pages/` のファイルがそのままルート
- **Content Collections** — `posts` / `talks` を MDX + Zod で型付け
- **ビルド時生成** — OGP（satori）/ RSS / sitemap / expressive-code
- **View Transitions** — スライド遷移

</div>
<div>

### React — Island で動かす

- **グラフ** — Highcharts（Treemap / Burndown / PV）
- **3D 背景** — three.js + R3F（ASCII / Orbital）
- **検索** — Fuse.js（PostSearch / TalkSearch）
- **外部データ** — GitHubActivity / PopularPosts
- **カウントダウン** — LifeProgress / CreativeLifeCountdown

</div>
</TwoColumn>

> Astro = 骨格 / React = 動く部品。お客さんがサッと見たい部分は軽量にできた。

## 学び② View Transitions の遷移は "一瞬の未確定 DOM" を通る

`/Talks`のスライド生成機構を作ったときの話

<TwoColumn>
<div>

### 始めたときの認識

- スライド機能を作る途中、トップページで使っている ASCII 装飾をスライドにも入れたい！
- 学び① のノリで「一旦置いて、後で詰めればいける」と思った

</div>
<div>

### 実際に起きたこと

- 1 スライド目はちゃんと動く
- View Transition で別スライドに遷移すると **`getImageData: Value is not of type 'long'` で drei がクラッシュ。遷移の瞬間だけ throw する**

</div>
</TwoColumn>

<SlideFootnote slot="footnote">
**drei** — R3F のヘルパーライブラリ集（`@react-three/drei`）／**Canvas** — Three.js シーンを描画する土台要素／**throw** — JavaScript で例外を投げる動作。投げた瞬間に実行が止まる
</SlideFootnote>

<Notes>
- これ実は Astro 固有じゃなくて、VT 全般で起きる話なんですが、Astro 経由で気付いたので載せておきました。
- 「トップで動かしている ASCII（Section ①② の裏でも動いていたあれ）をスライドにも入れたい話」と入る
- 「学び① の "後で詰める" 戦法、今回は通用しなかった」と振って次へ

</Notes>
</Slide>

<Slide>

<TwoColumn>
<div>

### 原因になっていた VT の挙動

1. VT は遷移中、古い DOM と新しい DOM が一瞬同居する
2. 新しい DOM が出てきた直後、Canvas は「幅 0」のフレームを経由する
3. そこで React Island がマウントされ、drei が `getImageData` を呼ぶと throw

**「マウント時 ＝ サイズが確定した DOM」という前提が、VT の遷移中だけ崩れる**

</div>
<div>

### LLM と粘った結果

試した順: rAF 1 つ挟む → VT 完了を待つ → `ResizeObserver`

- **着地点** — `ResizeObserver` で Canvas の実寸が出た瞬間にマウント
- **保険** — drei が出す一回限りのエラーは window で `preventDefault`

同じことやる人がいたら参考に。

</div>
</TwoColumn>

<SlideFootnote slot="footnote">
drei — R3F のヘルパーライブラリ集（`@react-three/drei`）／ **Canvas** — Three.js シーンを描画する土台要素 ／ **throw** — JavaScript で例外を投げる動作。投げた瞬間に実行が止まる ／ **rAF** — `requestAnimationFrame`。次の描画フレームまで処理を 1 つ遅らせるブラウザ API
</SlideFootnote>



<Notes>
- 「VT は便利、でも遷移中の一瞬を切り取ると DOM 状態は不安定」を強調
- 「AI と一緒に試行錯誤して着地した」を一言触れる
</Notes>
